# إعداد Telegram Bot لـ Army81

## الخطوة 1: إنشاء البوت

1. افتح Telegram وابحث عن **@BotFather**
2. أرسل: `/newbot`
3. اختر اسماً: `Army81 Assistant`
4. اختر username: `army81_bot` (أو أي اسم فريد ينتهي بـ `_bot`)
5. سيعطيك BotFather **Token** — احفظه

## الخطوة 2: الحصول على Chat ID

1. ابحث عن **@userinfobot** في Telegram
2. أرسل له `/start`
3. سيعطيك رقمك (Chat ID) — احفظه

## الخطوة 3: إضافة المفاتيح

عدّل ملف `.env`:

```
TELEGRAM_BOT_TOKEN=123456:ABCdef...   # Token من BotFather
TELEGRAM_CHAT_ID=987654321             # رقمك من userinfobot
```

## الخطوة 4: التشغيل

### محلياً:
```bash
python integrations/telegram_bot.py
```

### بـ Docker:
```bash
docker-compose up -d telegram_bot
```

## الخطوة 5: الاختبار

اذهب للبوت على Telegram وأرسل `/start`

يجب أن ترى رسالة ترحيب من Army81.

## الأوامر المتاحة

| الأمر | الوظيفة |
|-------|---------|
| `/start` | رسالة ترحيب |
| `/status` | حالة النظام |
| `/pending` | الطلبات المنتظرة موافقتك |
| `/report` | آخر تقرير يومي |
| `/agents` | قائمة الوكلاء |
| `/agent_A04 سؤالك` | سؤال وكيل محدد |
| أي نص | يُوجّه للوكيل المناسب تلقائياً |

## كيف يعمل

```
أنت (Telegram) → Bot → Gateway (8181) → SmartRouter → الوكيل المناسب → الرد
```

### التقارير التلقائية:
- **1:00 صباحاً** — اكتشاف معرفة جديدة
- **2:00 صباحاً** — تقرير يومي يُرسل لك
- **كل أحد** — تقرير التطوير الذاتي الأسبوعي

### الموافقات:
عندما يكتشف النظام شيئاً يحتاج موافقتك:
- يرسل لك رسالة مع أزرار ✅/❌
- تضغط الزر
- النظام ينفّذ أو يرفض

## استكشاف الأخطاء

| المشكلة | الحل |
|---------|------|
| Bot لا يستجيب | تأكد من صحة TOKEN في `.env` |
| "السيرفر غير متاح" | شغّل `python gateway/app.py` أولاً |
| لا تقارير | تأكد من `TELEGRAM_CHAT_ID` |
